# 提案：基金跟踪 v2——调仓 diff（相邻报告期持仓对比）

## Why

v1 已把每只关注基金的各报告期持仓按 `as_of_date` 版本化落库（含持股数），但 UI 只能看最新一期的静态前十大——**基金经理"改了什么"这个最有信号价值的信息**（新建仓/清仓/大幅加减仓）仍需人工对照两期表格。v2 把相邻版本对比做成一等功能，采用**股数口径优先**（权重口径会被股价涨跌污染，v1 提案已论证并为此保存了 `shares_wan`），为后续信号计算与抱团分析（v3）提供直接可用的变动事实。

## What Changes

- 新增后端调仓 diff 能力：对同一组合的两个版本（默认最新两期）计算逐股变动，分类为**新建仓 / 清仓 / 加仓 / 减仓 / 持平**，以持股数（万股）变化为主口径，权重变化（百分点）为辅助展示。
- **披露口径感知**（本变更的核心领域难点）：季报版本仅披露前十大（含 `*` 号行），中报/年报版本为全量。系统区分每个版本的披露口径（全量/部分），跨口径对比时使用「新进披露 / 退出披露（可能仍持有）」等诚实标签，**避免把"跌出前十大"误判为"清仓"**。
- 新增两个 IPC 命令：`get_portfolio_versions`（组合的版本列表，含各版本行数与口径）、`get_portfolio_diff`（两版本对比结果）。
- 前端基金组合卡片新增「调仓」视图：报告期选择（默认最新 vs 上一期）+ 分类变动明细表，按变动幅度排序。
- **刷新改为固定双年拉取**（当年+上一年，多 1 个请求）：否则年初到 Q2 季报披露前新增基金只有一期版本，调仓对比不可用；同时兑现 v1"历史可随时回补"的设计意图。
- diff 计算本身纯读库：**零 DB schema 变更、零新依赖**，不触碰启动流程。

明确不做（后续另立 change）：调仓信号阈值与提醒、跨基金抱团聚合、主动偏离度（基金权重−指数权重）、ETF PCF 接入、diff 结果导出。

## Capabilities

### New Capabilities

（无）

### Modified Capabilities

- `fund-tracking`: 新增调仓对比相关需求——版本列表查询、相邻版本 diff 计算（含披露口径语义与股数主口径）、调仓明细展示。

## Impact

- **后端**：新增 `src-tauri/src/services/position_diff.rs`（纯函数 diff 逻辑，可单测）；`commands/portfolios.rs` 追加两个只读命令；`lib.rs` 注册。
- **前端**：`src/types/index.ts` 增镜像类型；`portfolioStore` 增两个 invoke；`src/pages/Funds/` 卡片内增调仓视图组件。
- **数据库**：无 migration。披露口径在查询时由「版本行数 + 报告期季度」启发式推断，不落新列（v1 设计的既定预案）。
- **风险**：口径推断是启发式（行数阈值），对持仓极度集中的基金可能误判全量为部分——影响仅限标签措辞（偏保守方向），不产生错误的"清仓"断言。
